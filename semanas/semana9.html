1. Ventana de Ingreso Login con Roles

Implementación de un sistema de autenticación con tres tipos de usuarios: Estudiante, Profesor y Administrador.

Características del Sistema

El sistema de login implementado incluye validación de credenciales, manejo de sesiones y redirección según el rol del usuario.

Estructura de Roles

  • Estudiante: Acceso a recursos educativos y gestión personal
  • Profesor: Herramientas de evaluación y gestión académica
  • Administrador: Control total del sistema y gestión de usuarios

Implementación Java - Clase Login

public class LoginController {
    public boolean autenticarUsuario(String email, String password) {
        // Validación de credenciales en base de datos
        Usuario usuario = usuarioDAO.buscarPorEmail(email);
        
        if (usuario != null && 
            BCrypt.checkpw(password, usuario.getPassword())) {
            
            // Establecer sesión según rol
            HttpSession session = request.getSession();
            session.setAttribute("rol", usuario.getRol());
            session.setAttribute("nombre", usuario.getNombre());
            
            return true;
        }
        return false;
    }
    
    public void redirigirSegunRol(HttpServletResponse response, String rol) 
            throws IOException {
        switch(rol) {
            case "ESTUDIANTE":
                response.sendRedirect("estudiante.jsp");
                break;
            case "PROFESOR":
                response.sendRedirect("profesor.jsp");
                break;
            case "ADMIN":
                response.sendRedirect("admin.jsp");
                break;
        }
    }
}

Servlet - Gestión de Cursos

@WebServlet("/profesor/cursos")
public class GestionCursosServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, 
                         HttpServletResponse response) 
            throws ServletException, IOException {
        
        String profesorEmail = (String) request.getSession()
                                .getAttribute("email");
        List<Curso> cursos = cursoDAO.obtenerCursosPorProfesor(profesorEmail);
        
        request.setAttribute("cursos", cursos);
        request.getRequestDispatcher("/profesor/cursos.jsp")
               .forward(request, response);
    }
}

Características Principales

  • Componentes predefinidos: Navbars, cards, modals
  • Utilidades CSS: Margin, padding, colors, flexbox
  • JavaScript integrado: Componentes interactivos

Implementación en Maven

<!-- En pom.xml -->

Configuración del entorno de desarrollo para proyectos web con NetBeans y gestión de dependencias con Maven.

├── src/
│   └── main/

│       │       ├── controllers/
│       │       ├── models/

│       │   ├── profesor/

Configuración pom.xml

<project>
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.empresa</groupId>
    <artifactId>sistema-academico</artifactId>
    <version>1.0.0</version>
    <packaging>war</packaging>
    
    <dependencies>
        <!-- Servlet API -->
        <dependency>
            <groupId>javax.servlet</groupId>
            <artifactId>javax.servlet-api</artifactId>
            <version>4.0.1</version>
            <scope>provided</scope>
        </dependency>
        
        <!-- JSTL -->
        <dependency>
            <groupId>javax.servlet</groupId>
            <artifactId>jstl</artifactId>
            <version>1.2</version>
        </dependency>
        
        <!-- MySQL Connector -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <version>8.0.33</version>
        </dependency>
    </dependencies>
</project>
