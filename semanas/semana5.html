Semana 5 - Planificación del Sistema de Login

Fase de diseño y planificación previa al desarrollo del sistema de autenticación. Documentación de requisitos, arquitectura y características planeadas para la implementación.

Proceso de Desarrollo Planificado

Análisis

Identificación de requisitos y definición de funcionalidades del sistema de login

Diseño

Creación de interfaces, definición de arquitectura y planificación de componentes

Desarrollo

Implementación del código con validaciones, seguridad y experiencia de usuario

Pruebas

Validación de funcionalidades, seguridad y experiencia del usuario final

Objetivos del Sistema

Definición de los propósitos principales y requisitos funcionales que debe cumplir el sistema de autenticación.

Autenticación de Usuarios

  • Validación de credenciales (usuario y contraseña)
  • Manejo de diferentes roles (estudiante, profesor)
  • Control de sesiones de usuario
  • Recuperación de contraseña (futura implementación)

Seguridad

  • Límite de intentos fallidos (3 intentos máximo)
  • Encriptación de contraseñas en la base de datos
  • Validación de entrada contra inyecciones SQL
  • Manejo seguro de sesiones

Experiencia de Usuario

  • Interfaz intuitiva y responsive
  • Feedback visual inmediato en validaciones
  • Placeholders dinámicos en campos de entrada
  • Navegación fluida entre login y registro
  • Tiempo de autenticación: Menos de 2 segundos
  • Disponibilidad: 99% del tiempo operativo
  • Seguridad: 0 vulnerabilidades críticas
  • Usabilidad: 95% de usuarios satisfechos
  • Rendimiento: Soporte para 100+ usuarios concurrentes

Implementación FrontEnd

Desarrollo de las interfaces gráficas del sistema de login utilizando Java Swing con arquitectura MVC.

Descripción del Desarrollo

Se implementaron las interfaces gráficas principales del sistema de login utilizando Java Swing. El sistema incluye formularios de login, registro y menú principal, siguiendo los principios de la arquitectura MVC y aplicando buenas prácticas de desarrollo.

Características Implementadas

Las interfaces desarrolladas incluyen validaciones de entrada, navegación entre formularios, manejo de eventos de usuario y una estructura modular que facilita el mantenimiento y escalabilidad del sistema.

FormLogin.java - Formulario Principal de Login

package com.mycompany.loginn;

import Clases.CLogin;

public class FormLogin extends javax.swing.JFrame {
    public FormLogin() {
        initComponents();
    }

    private void jButton1ActionPerformed(java.awt.event.ActionEvent evt) {
        // Validación de usuario
        Clases.CLogin objetoLogin = new Clases.CLogin();
        objetoLogin.validaUsuario(txtUsuario, txtContrasenia);
    }

    private void jButton2ActionPerformed(java.awt.event.ActionEvent evt) {
        // Navegación a formulario de registro
        FormSignUp SignupFrame = new FormSignUp();
        SignupFrame.setVisible(true);
        SignupFrame.pack();
        SignupFrame.setLocationRelativeTo(null);
        this.dispose();
    }
}
