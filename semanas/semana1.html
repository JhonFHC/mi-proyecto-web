Control de Versiones con GIT

GIT es un sistema de control de versiones distribuido que permite gestionar el historial de cambios de un proyecto, facilitando el trabajo en equipo y el mantenimiento del código.

GIT guarda un historial de versiones llamado Snapshot o Commit, que permite almacenar solo los cambios realizados. Esto facilita trabajar sin conexión y sincronizar después con repositorios remotos.

  • Working Directory: Archivos editables del proyecto
  • Staging Area: Cambios seleccionados para guardar
  • Repositorio: Lugar donde se almacenan las versiones

Una rama permite separar el desarrollo para probar ideas sin afectar la rama principal, pudiendo luego unir los cambios mediante merge.

  • main/master: Rama principal del proyecto
  • develop: Rama de desarrollo
  • feature: Ramas para nuevas funcionalidades
  • hotfix: Ramas para correcciones urgentes

Comandos Básicos de GIT

# Inicializar repositorio
git init

# Clonar repositorio existente
git clone [url]

# Agregar cambios al staging
git add .

# Crear commit
git commit -m "mensaje"

# Subir cambios al repositorio remoto
git push origin main

Apache Maven

Maven es una herramienta de gestión y construcción de proyectos que simplifica el proceso de build, dependency management y documentación.

  • Gestión de Dependencias: Descarga automática de librerías
  • Construcción Uniforme: Mismo proceso para todos los proyectos
  • Gestión del Ciclo de Vida: Compilación, testing, packaging
  • Arquetipos: Plantillas para crear proyectos rápidamente

El archivo POM (Project Object Model) es el corazón de Maven, donde se define la configuración del proyecto.

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.miempresa</groupId>
  <artifactId>mi-proyecto</artifactId>
  <version>1.0.0</version>
  
  <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.13.2</version>
    </dependency>
  </dependencies>
</project>
# Compilar el proyecto
mvn compile

# Ejecutar tests
mvn test

# Empaquetar la aplicación
mvn package

# Instalar en repositorio local
mvn install

# Crear proyecto desde arquetipo
mvn archetype:generate

Descripción del Problema

Durante la clase, se presentó un problema relacionado con la configuración de dependencias en Maven y la gestión de versiones de librerías. El problema consistía en resolver conflictos entre diferentes versiones de dependencias transitivas y optimizar la estructura del archivo POM.xml para un proyecto Java.

Solución Implementada

La solución incluyó el uso de la etiqueta <dependencyManagement> para centralizar la gestión de versiones, la exclusión de dependencias conflictivas mediante <exclusions>, y la implementación de plugins específicos para optimizar el proceso de build.

Fragmento de la Solución

<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-dependencies</artifactId>
      <version>2.7.0</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>
